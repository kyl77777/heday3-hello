# AI 코드 어시스턴스 교육 프로그램 판매 사이트 설계서

## 1) 개요
- 목표: 40~60대 장년층을 대상으로 오프라인 대면 수업 기반 AI 코드 어시스턴스 교육 프로그램의 상담 신청 전환을 극대화한다.
- 핵심 가치: 비개발자도 따라오는 쉬운 커리큘럼
- 톤앤매너: 성과·동기부여 중심

## 2) 정보 구조(단일 랜딩 페이지)
정적 웹사이트의 단일 페이지(`index.html`)로 구성한다.

1. 히어로 섹션
   - 성과 중심 헤드라인
   - 서브카피(비개발자 친화 메시지)
   - 상담 CTA 3종(전화, 카카오, 문의하기)
2. 공감 문제 제시 섹션
   - 장년층 학습 장벽(용어, 속도, 실습 불안) 제시
   - 해결 메시지 연결
3. 커리큘럼 섹션
   - 운영 정보: 8주, 주 2회, 회당 2시간
   - 주차별 성과 중심 설명
4. 오프라인 강점 섹션
   - 현장 질문 즉시 해결
   - 소규모 밀착 코칭
   - 실습 중심 진행
5. 신뢰 섹션
   - 강사 소개
   - 수강생 변화 사례
   - FAQ
6. 안내 섹션
   - 수강료: 120만 원(상담 시 분할 안내 가능)
   - 위치: 서울 강남역 인근(도보 5분)
7. 상담 전환 섹션(반복 배치)
   - 전화 버튼
   - 카카오 상담 버튼
   - 문의 폼

## 3) 전환 경로 설계
- 전화 CTA: `tel:` 링크로 즉시 통화 연결
- 카카오 CTA: 카카오 상담 채널 링크로 이동
- 문의 폼: 이름/연락처/관심과정 입력 후 외부 폼 백엔드로 제출
  - 정적 사이트 특성상 Formspree(또는 동급 서비스) POST 연동
- 제출 성공: 24시간 내 연락 안내 메시지 + 주요 CTA 재노출
- 제출 실패: 오류 메시지 + 재시도 유도

## 4) 컴포넌트 정의
- 고정 상단 CTA 바
  - 스크롤 중에도 항상 상담 진입 가능
- 히어로 블록
  - 헤드라인, 서브카피, 핵심 CTA 3종
- 커리큘럼 카드
  - 단계별 난이도 완화, 실습 결과 중심 설명
- 오프라인 강점 카드
  - 대면 학습의 즉시 피드백 강조
- 신뢰 카드
  - 후기/사례/강사 신뢰 요소
- 문의 폼
  - 필수 입력 + 제출 상태 피드백

## 5) 폼 정책 및 오류 처리
- 필수값
  - 이름: 필수
  - 연락처: 필수(숫자/하이픈 허용)
  - 관심과정: 필수
- 오류 노출
  - 필드별 인라인 메시지
  - 제출 실패 시 상단 공통 메시지
- 성공 노출
  - 제출 완료 메시지 + 후속 안내 문구

## 6) 접근성/사용성 기준
- 라벨과 입력 필드 연결
- 충분한 명도 대비
- 키보드 포커스 이동 보장
- 모바일 터치 영역 충분히 확보

## 7) 콘텐츠 원칙
- 장년층 관점의 쉬운 문장 사용
- 전문용어 최소화, 필요한 경우 즉시 풀이
- 과장보다 구체적 성과 사례를 우선
- 모든 핵심 섹션에 상담 CTA를 반복 배치

## 8) 검증 기준(수동 테스트)
1. 전화 링크 클릭 시 통화 앱 연결 확인
2. 카카오 링크 정상 이동 확인
3. 문의 폼 유효/무효 입력 검증 확인
4. 제출 성공/실패 메시지 분기 확인
5. 모바일 화면에서 글자 가독성 및 버튼 터치성 확인
6. 스크롤 전 구간에서 상담 CTA 접근성 확인

## 9) 범위
- 포함
  - 단일 정적 랜딩 페이지 설계
  - 상담 전환 중심 구조
  - 외부 폼 서비스 연동 전제
- 제외
  - 결제 시스템 구현
  - 관리자 페이지 구현
  - 회원 인증/로그인
